## 题目

> leetcode 链接： [字符串转换整数 (atoi)](https://leetcode.cn/problems/string-to-integer-atoi/)

请你来实现一个 myAtoi(string s) 函数，使其能将字符串转换成一个 32 位有符号整数（类似 C/C++ 中的 atoi 函数）。

函数 myAtoi(string s) 的算法如下：

1. 读入字符串并丢弃无用的前导空格
2. 检查下一个字符（假设还未到字符末尾）为正还是负号，读取该字符（如果有）。 确定最终结果是负数还是正数。 如果两者都不存在，则假定结果为正。
3. 读入下一个字符，直到到达下一个非数字字符或到达输入的结尾。字符串的其余部分将被忽略。
4. 将前面步骤读入的这些数字转换为整数（即，"123" -> 123， "0032" -> 32）。如果没有读入数字，则整数为 0 。必要时更改符号（从步骤 2 开始）。
5. 如果整数数超过 32 位有符号整数范围 [−2^31, 2^31 − 1] ，需要截断这个整数，使其保持在这个范围内。具体来说，小于 −2^31 的整数应该被截断为 −2^31 ，大于 2^31 − 1 的整数应该被截断为 2^31 − 1 。
6. 返回整数作为最终结果。

**注意：**
- 本题中的空白字符只包括空格字符 ' ' 。
- 除前导空格或数字后的其余字符串外，请勿忽略任何其他字符。

**示例 1：**

```
输入：s = "42"
输出：42
解释：加粗的字符串为已经读入的字符，插入符号是当前读取的字符。
第1步："42"（当前没有读入字符，因为没有前导空格）
         ^
第2步："42"（当前没有读入字符，因为这里不存在 '-' 或者 '+'）
         ^
第3步："42"（读入 "42"）
           ^
解析得到整数 42 。
由于 "42" 在范围 [-2^31, 2^31 - 1] 内，最终结果为 42 。
```

```
             ^
第3步："   -42"（读入 "42"）
               ^
```


## 题解

**题意简化：**

这道题就像我们小时候做的“把一串字符变成数字”的游戏。
1.  首先，我们要忽略字符串前面的空格。
2.  然后，我们要判断字符串的第一个非空字符是不是加号`+`或者减号`-`，来确定结果是正数还是负数。
3.  接着，我们要把字符串中连续的数字字符转换成数字。
4.  如果转换后的数字超出了 `[-2^31, 2^31 - 1]` 这个范围，我们要把它截断到这个范围的边界值。
5.  如果字符串不是以数字开头，或者根本没有数字，那么结果就是 0。

**边界值与测试用例：**

*   空字符串 `""`，结果应为 `0`
*   只有空格的字符串 `" "`，结果应为 `0`
*   以非数字字符开头的字符串 `"abc"`，结果应为 `0`
*   包含正负号的字符串 `"+123"`，`"-123"`
*   超出范围的正数 `"2147483648"`，结果应为 `2147483647`
*   超出范围的负数 `"-2147483649"`，结果应为 `-2147483648`
*   正常范围内的数字 `"123"`, `"-42"`,`"4193 with words"`

测试用例：

| 输入               | 输出         |
| ------------------ | ------------ |
| `""`               | `0`          |
| `"   "`            | `0`          |
| `"abc"`            | `0`          |
| `"+123"`           | `123`        |
| `"-123"`           | `-123`       |
| `"2147483648"`     | `2147483647` |
| `"-2147483649"`    | `-2147483648`|
| `"123"`            | `123`        |
| `"-42"`            | `-42`        |
| `"4193 with words"`| `4193`       |

**解法分析：**

| 解法           | 优点                               | 缺点                                 | 时间复杂度 | 空间复杂度 |
| -------------- | ---------------------------------- | ------------------------------------ | -------- | -------- |
| 遍历字符串     | 直观易懂                           | 需要处理多种情况，代码稍显冗长         | O(n)     | O(1)     |
| 正则表达式     | 代码简洁                           | 正则表达式的效率可能较低               | 依赖正则引擎 | 依赖正则引擎 |

复杂度评估方法：

*   时间复杂度：主要取决于遍历字符串的长度。
*   空间复杂度：只使用了常数级别的额外空间。

推荐解法：

我推荐使用**遍历字符串**的方法，因为它最容易理解和记忆，也适合在面试中快速实现。

**Python代码实现：**

版本一：详细注释

```python
class Solution:
    def myAtoi(self, s: str) -> int:
        # 1. 去除前导空格
        s = s.lstrip()
        
        # 2. 判断正负号
        sign = 1
        if s and (s[0] == '+' or s[0] == '-'):
            if s[0] == '-':
                sign = -1
            s = s[1:]  # 去掉符号位
        
        # 3. 读取数字
        result = 0
        for char in s:
            if char.isdigit():
                digit = int(char)
                
                # 4. 检查是否溢出
                if result > (2**31 - 1) // 10 or (result == (2**31 - 1) // 10 and digit > 7):
                    return 2**31 - 1 if sign == 1 else -2**31
                
                result = result * 10 + digit
            else:
                break  # 遇到非数字字符，停止读取
        
        # 5. 应用正负号并返回
        return sign * result
```

```python
class Solution:

        s = s.lstrip()

```

**代码运行过程详细描述：**

1.  **去除前导空格：** `s = s.lstrip()`  移除字符串 `s` 开头的空格。
2.  **判断正负号：** 检查字符串的第一个字符是否为 `+` 或 `-`。如果是 `-`，则将 `sign` 设置为 `-1`。
3.  **读取数字：** 遍历字符串，如果字符是数字，则将其转换为整数，并加到 `result` 中。
4.  **检查溢出：** 在每次将数字加到 `result` 中之前，检查是否会超出 32 位有符号整数的范围。如果超出，则返回最大或最小值。
5.  **应用正负号：** 将 `result` 乘以 `sign`，得到最终结果。
